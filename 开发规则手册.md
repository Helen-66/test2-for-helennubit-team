# 开发规则手册

## 分支管理

- 主分支为 `main`，不允许直接提交
- 新功能或修复请从 `main` 创建新分支
- 分支命名规范：`feature/xxx`、`fix/xxx`、`docs/xxx`

## 代码规范

- 使用 4 空格缩进
- 变量和函数命名使用小写加下划线（snake_case）
- 每个函数应有清晰的职责，避免过长

## 提交规范

- 提交信息格式：`<type>: <description>`
- 常用类型：feat、fix、docs、refactor、test
- 提交前确保代码无语法错误

## Code Review

- 所有代码需通过 PR 合并到 main
- PR 至少需要一位团队成员审核
- 审核通过后方可合并

## 安全要求

- 禁止在代码中硬编码密钥或密码
- 敏感配置使用环境变量管理
